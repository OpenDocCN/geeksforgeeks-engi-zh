# DMAIC 和 DMADV 的区别

> 原文:[https://www . geesforgeks . org/dmaic 和-dmadv 之间的区别/](https://www.geeksforgeeks.org/difference-between-dmaic-and-dmadv/)

前提条件:[软件工程六适马](https://www.geeksforgeeks.org/six-sigma-in-software-engineering/)

**1。DMAIC :**

DMAIC 是六西格玛项目的一部分，代表定义、测量、分析、改进和控制。DMAIC 是一种商业策略，用来找出如何在控制成本的同时改进流程。这是一种有效的方法，可以识别运营或周围环境中重大问题的根本原因，然后利用这些知识来制定解决方案。

1.  **定义:**首先确定自己心中对自己想要改进的东西有一个目标，尽可能的理解问题；这将有助于确定正在改进的流程。
2.  **测量:**接下来建立一个基线，在进行任何更改之前，您可以通过收集数据来测量改进。这可能涉及制作控制图或从过去的项目/工作中收集数据，因为它包含类似的问题组。
3.  **分析:**等你积累好数据后，分析一下看能做些什么改进，得到一个基线图。这是使用控制图和帕累托图来查看正在发生哪些改进以及哪些需要进一步改进的好时机。
4.  **改善:**在这个阶段，你应该开始做出改变，这将改善你的过程。其中最重要的一个因素是在改掉坏习惯和创造新的好习惯方面改变系统思维。DMAIC 周期的这一部分是为了公司的持续改进和系统的整体健康。
5.  **控制:**完成所有改进后，再次查看数据，看看变化有多大，数据是否有波动或趋势。这将向您展示您是否需要做更多或更少您已经实现的事情。

**2.DMADV：**

DMADV 是六西格玛项目的一部分，代表定义、测量、设计、分析和验证。这些是设计过程中的通用工具，我们发现这些工具有助于进一步了解项目的进展情况。

这五个阶段是:

1.  **定义:**定义阶段是你对如何执行设计建立共识的阶段。
2.  **测量:**测量阶段是你收集数据，帮助你了解你的项目是如何展开的。
3.  **设计:**设计阶段是你利用收集到的所有信息，以便对你可能采取的下一步措施做出明智的决定。已经收集的信息可以被视为推动项目取得更好结果的机会，而不是风险的来源。
4.  **分析:**分析阶段是你把第 2 步和第 3 步收集到的所有信息，对其进行分析，帮助你了解你的项目到底发生了什么。
5.  **验证:**验证阶段是当你比较期望项目如何展开和项目实际如何展开的时候。通过做这个比较，你正在验证你最初的意图正在被满足。

**什么时候用？**

1. **DMAIC :** 当产品已经发布到公司之外，但是存在问题或者产品偏离客户需求时，使用 DMAIC 方法论。

2. **DMADV :** 当产品未建成且目前处于设计&规划过程中时，使用 DMADV 方法论。当产品无法优化并且需要重建时，也可以使用它。

**DMAIC 和 DMADV 的区别:**

<figure class="table">

| **序列号** | **DMAIC** | **DMADV** |
| 01. | DMAIC 是关于改进和控制过程的。它定义了业务流程。找到问题，然后以提供改进的解决方案为目标来解决它。 | 设计和验证过程包括重新设计整个过程，以接近客户描述的要求。 |
| 02. | 它解决了项目的当前流程。 | 它解决了项目的设计过程。 |
| 03. | 它能更好地与现有项目配合使用。 | 它更适合新项目。 |
| 04. | 它是关于最小化过程和纠正错误。 | 而它是关于防止错误的。 |
| 05. | 它使用定量工具。 | 它使用定性工具 |
| 06. | DMAIC 通常被认为是短期项目。 | DMADV 通常被认为是长期项目。 |
| 07. | DMAIC 给出了具体的解决方案。 | DMADV 不是整个解决过程，它是设计解决方案的一部分。 |
| 08. | 这是关于纠正。 | 这是关于预防。 |
| 09. | DMAIC 控制着未来的工艺性能。 | DMADV 验证了设计性能。 |
| 10. | 它是从一个问题开始的。 | 而它是从创新解决方案开始的。 |

</figure>
