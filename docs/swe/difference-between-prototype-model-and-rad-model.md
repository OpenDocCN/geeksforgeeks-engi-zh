# 原型模型和 RAD 模型的区别

> 原文:[https://www . geesforgeks . org/原型-模型-和-rad-模型的区别/](https://www.geeksforgeeks.org/difference-between-prototype-model-and-rad-model/)

**1。[原型模型](https://www.geeksforgeeks.org/software-engineering-prototyping-model/) :**
原型模型是一个[软件开发生命周期模型](https://www.geeksforgeeks.org/what-is-sdlc-model-and-its-phases/)，在客户对最终产品应该是怎样的以及它的需求还不完全了解的时候使用。因此，在这个模型中，最终产品的原型首先由开发人员开发，然后进行测试，并根据客户反馈进行更改，直到客户对原型满意为止。

**2。 [RAD 模型](https://www.geeksforgeeks.org/software-engineering-rapid-application-development-model-rad/) :**
RAD 模型是指快速应用开发，属于[增量模型](https://www.geeksforgeeks.org/software-engineering-incremental-process-model/)的类型。组件或功能是并行开发的，可以处理小型项目，也可以处理中型项目。在这个模型中，任何变化都可以在任何阶段进行。因为人少，所以生产力高。

**原型模型与 RAD 模型的区别:**

<center>

| 没有。 | 原型模型 | RAD 模型 |
| 1. | 原型模型是一种软件开发模型，其中原型被构建、测试，然后根据客户需求进行改进。 | Rad 模型是一种软件开发模型，其中组件或功能被并行开发，就像它们是小型项目一样。 |
| 2. | 适用于高风险项目。 | 技术风险高的时候不适合。 |
| 3. | 它没有强调风险分析。 | 它强调风险分析。 |
| 4. | 在这种情况下，快速的初步审查是不可能的。 | 在这种情况下，快速的初步审查是可能的。 |
| 5. | 它不支持自动代码生成。 | 它支持自动代码生成。导致最少的代码编写。 |
| 6. | 它最适合于客户的需求不明确并且应该改变的情况。 | 它最适合客户要求明确的情况。 |
| 7. | 在这方面，用户参与度很高。 | 在这种情况下，用户参与只是开始。 |

</center>
