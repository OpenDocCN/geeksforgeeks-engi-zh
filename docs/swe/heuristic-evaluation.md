# 启发式评估

> 原文:[https://www.geeksforgeeks.org/heuristic-evaluation/](https://www.geeksforgeeks.org/heuristic-evaluation/)

**启发式评估的需求:**

[启发式评估](https://www.geeksforgeeks.org/difference-between-usability-testing-and-heuristic-evaluation/)是一个彻底评估/评价的过程，其中特定领域的专家用于衡量用户界面的可用性。可用性可以被定义为一个特定的用户可以多么容易地使用一个特定的设计或者说界面而不面临任何问题。一般来说，我们可以说执行启发式评估是为了检测产品设计中的问题。它还确定了解决设计中存在的问题并满足用户期望的方法。

启发式评估是由专家执行的深入可用性测试。众所周知，可用性越好，与产品交互的用户数量就越多。Jakob Nielsen 和 Rolf Molich 是网络可用性的先驱，他们在 1990 年发表了这篇文章，其中包含了一套启发式方法。启发式可以被定义为快速和实用的方法来处理问题，并做出有效的决策来解决这些问题。专家使用启发式方法系统地评估用户体验(UX)设计。

**何时进行启发式评估:**

何时执行启发式评估没有这样的规则，但是它可以在设计过程的任何阶段执行。大多数时候，启发式评估是在论文原型和可用性测试之后进行的。由于启发式评估有助于优化用户界面的设计，因此评估最终设计变得非常重要。

**如何进行启发式评估:**

**定义评估范围–**
在评估时提及预算和截止日期变得非常重要。还应该定义他们想要进行可用性测试的不同参数。

**了解最终用户–**
众所周知，不同的人群对一款产品有不同的期待。因此，了解最终用户及其兴趣变得非常重要。

**选择你的试探法集–** **如果没有合适的试探法，如果所有的评估者不使用相同的指导方针，试探法评估将产生不可靠和无用的结果。**

****建立评估系统并识别问题–**
确定问题的不同类别，如关键问题、次要问题等。评估者必须遵循系统评估的指导方针。**

****分析和总结结果–**
分析用户界面设计中存在的问题并在截止日期前解决这些问题变得非常必要。**

****优势:****

*   **揭示了许多隐藏的可用性问题。**
*   **它有助于确定整体用户体验。**
*   **启发式评估可以与可用性测试相结合。**
*   **更好的启发式评估有助于吸引更多用户。**
*   **这比进行全面的可用性测试更便宜、更快。**

****缺点:****

*   **有时候，即使是专家也很难弄清楚一些问题。**
*   **很难找到专家来进行启发式评估。**
*   **我们将需要很少的专家评估者，这样我们将更容易坚持可用性测试。**
*   **设计上的缺陷会影响用户对产品的参与度。**
*   **启发式测试只依赖于少数专家的专业水平。**
